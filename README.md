Project EduNexus：校园超算可视化中枢 - 功能描述说明书
1. 项目概述
Project EduNexus 是一款颠覆传统教务管理系统的未来风桌面级应用。本系统秉持“万物皆可算”的理念，将校园日常运营中的复杂行政、教务问题（如排课、选课、路线规划、查重等），转化为经典的计算机科学与信息学奥赛（OI）硬核算法模型。

在提供极高计算精度的同时，系统采用赛博朋克与超算中心风格的 UI 设计，将底层的“黑盒算法”以极具视觉冲击力的实时动态图表呈现在屏幕上，打造出“掌控全局数据洪流”的沉浸式体验。

2. 核心功能模块
模块一：量子退火排班排座中枢 (Quantum Seating & Scheduling)
业务功能： 解决包含身高、视力、同桌关系绑定、极度厌恶同桌规避、连堂课约束等极度复杂条件下的座位表与课表自动生成。

算法内核： 多目标模拟退火算法 (Simulated Annealing) / DLX 精确覆盖。

视觉呈现： 界面中心为一个动态能量矩阵图。算法启动时，座位网格呈现混乱且高频闪烁的红色警告状态；随着界面左侧“系统温度”进度条的逐渐冷却，网格的闪烁频率降低，冲突项被逐步消灭，最终结晶定格为整齐的青色最优解排布。

模块二：选课与资源博弈流网络 (Game Flow Network)
业务功能： 处理全校规模的选修课抢课、社团人员分配问题。系统根据学生的志愿优先级和各课程/社团的容量上限，计算出全局满意度最高的最优分配方案，避免传统先到先得的拥堵崩溃。

算法内核： 图论 - 最小费用最大流模型 (MCMF)。

视觉呈现： 构建一个赛博朋克风格的二分图（Bipartite Graph）。左侧代表学生节点，右侧代表课程节点。运算时，可视化的“数据水流”从左侧涌向右侧，网络连线随着分配流量的增加逐渐变宽并爆发出霓虹紫色，直观展示极其庞大的供需匹配推流过程。

模块三：校园动态巡航推演仪 (Patrol Path Finder)
业务功能： 针对重大考试的巡考老师路线规划，或校园夜间安保的重点区域巡逻规划。在保证覆盖所有必经节点的前提下，推演出耗时最短、路程最少的绝对最优路线。

算法内核： 旅行商问题 (TSP) - 状态压缩动态规划 / 蚁群算法。

视觉呈现： 界面载入 2.5D 的暗黑色调校园节点拓扑图。推演开始时，无数根半透明的探针光线从起点爆发，向全图蔓延（模拟寻路搜索）；右上角仪表盘疯狂滚动状态转移次数；最终，一条闪耀着金色流光的最优闭合回路将锁定在地图上。

模块四：天网学术查重矩阵 (Skynet Plagiarism Matrix)
业务功能： 极速完成全校信息技术课代码作业、或文科类大段作文的内部雷同率筛查。

算法内核： 高级字符串处理 - 多模式串匹配 AC 自动机 / 后缀自动机 (SAM)。

视觉呈现： 采用类似《黑客帝国》的代码瀑布流设计。被比对的文档在屏幕两侧以暗色调展示，一根醒目的红色“扫描光束”以极速扫过文本。一旦捕获雷同片段，涉事代码块瞬间被高亮为警报红，并在屏幕两端拉出锁定连线，同时弹出刺眼的 Match Rate 匹配率悬浮窗。

模块五：全息学情边界雷达 (Convex Hull Radar)
业务功能： 突破常规的平均分分析，将学生的多维度成绩转化为多维空间数据，精准捕捉年级中的“极端偏科奇才”或界定特定班级的综合实力边界极限。

算法内核： 计算几何 - Graham Scan 极角排序与凸包生成。

视觉呈现： 在带有极坐标背景的深色画布上，数百名学生的数据化身为呼吸闪烁的散点。激活扫描后，一条青色的雷达扫描线顺时针扫过整个星云状的数据集群，最外围的“边界学生”节点被逐一锁定点亮，并拉出一条动态的发光多边形边界线，点击边缘节点即可全息展开该名“奇才”的详细画像。

模块六：校园资源极速抢占系统 (Resource Knapsack Allocator)
业务功能： 在有限的条件内（如有限的社团活动总经费、有限的学生选课总学分），计算出如何组合各项活动或课程，才能达到总价值/总收益的最大化。

算法内核： 动态规划 - 01 背包 / 多重背包状态转移。

视觉呈现： 将抽象的 DP 数组转化为屏幕下方巨大的“状态转移栅格矩阵”。计算时，栅格阵列随波浪闪烁；计算完成瞬间，一条高亮电流从矩阵右下角的“最终解”开始逆流而上（Traceback），沿途点亮最终被纳入最优解方案的资源图标。

3. 总体视觉与交互规范 (UI/UX)
全景暗黑风格： 抛弃传统操作系统的亮色窗口，采用全局深邃黑（#090D14）底色，搭配无边框沉浸式视口。

霓虹高亮与亚克力材质： 大量使用极光青（#00FFCC）与霓虹紫（#B026FF）作为核心点缀色；功能面板采用大面积的玻璃态模糊（Glassmorphism）效果。

全局响应式终端： 界面底部常驻一个类似黑客控制台的终端日志窗口，实时打印底层的算法推演日志与耗时（如：[MCMF_Core] Path augmented, Flow +1, Time: 12ms），将极客氛围拉满。
